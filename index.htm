Salesforce Breach

The vast list of companies with customer support data compromised in the recent Salesforce Drift compromise is growing and includes big name brands Google, Palo Alto Networks, Cloudflare and Tenable. Cloudflare has provided a very comprehensive breakdown of the incident from their discovery through remediation. In a refreshingly transparent post we can see from Cloudflare's investigation a sustained probing of systems by the criminals for around ten days.

Salesloft commented on the 26th August: "Initial findings have shown that the actorâ€™s primary objective was to steal credentials, specifically focusing on sensitive information like AWS access keys, passwords, and Snowflake-related access tokens. We have determined that this incident did not impact customers who do not use our Drift-Salesforce integration. Based on our ongoing investigation, we do not see evidence of ongoing malicious activity related to this incident."

Given the nature of the information stolen, however, it is more than likely the details will make their way onto the open market or be utilised for phishing or targeted campaigns against customers of the brands.

Drift, the AI chat agent tooling, has been taken offline for now (one does wonder if it will return and how many people will trust it going forward). It's vital that anyone having utilised it in their workflows disconnects, rotates API tokens and runs forensic analysis on their data systems.

Computers help humanity make terrible decisions incredibly quickly, and AI tooling, it would appear, continues to prove that it makes it more effectively and terribly than a junior developer with no world experience.

Fake NPM Package Injects Code

An NPM package impersonating a legitimate email library has been found by researchers. The package has thankfully now been removed from NPM with a warning.

The package, nodejs-smtp, had an identical tagling, page style and README to the legitimate nodemailer package and has been available since April 2025 garnering 347 downloads since it was added to the npm registry.

The package uses Electron tooling to unpack Atomic Wallet's app.asar, replacing the vendor bundle with a malicious payload before repackaging the application and removing process traces by deleting the working directory. The primary objective of this is to overwrite the recipient address with hard-coded wallets redirecting Bitcoin, Ethereum, Tether, XRP and Solana transactions.

Still operating as a mailer, it hides the underlying functionality to swipe cryptocurrency in normal operation as an SMTP maailer.

It's not the first time we've seen issues with downstream pacakages in this type of attack and it won't be the last.

South Korean Telecom Fined for terrible security

One for the scrapbook of "what in the hell, my good man?", SK Telecom have mismanaged their network systems security so utterly terribly that the watchdog has fined them $97M for their sloppy practices. The breach, disclosed back in April, led to the unauthorised access to data for almost 27 million subscribers. That's over half the population of South Korea (50M). Over HALF the population!

According to the regulator's report, SK Telecom failed at every layer of defence and effectively rolled-out the red carpet to their own systems for the cyber criminals that ran amok through their network.

Administrators had left thousands of server credentials in plaintext on a management server; there was no password protection on subscriber databases; IDS logs were not reviewed by teams; 26 Million USIM authentication keys were left unencrypted in the database (leading to the ability to clone SIMS without knowledge); And no response from monitoring teams on the installation of malware and exfiltration of data.

The regulator provided a scathing observation "The security operating environment between the internet and the internal network was managed and operated in a state that was very vulnerable to illegal intrusion". Read that as corporate speak for "step away from the computer, Ralf Wiggum, it's time to go eat crayons"

Not only slapped with a hefty fine, SK Telecom have now been ordered to implement a range of remedial items including proper encryption, tighter access controls and real-time monitoring of IDS. Just a few things that any Telco should have been implementing by default, you would have imagined.

Telecommunication companies are a high value target that we often only see nation-backed or heavyweight groups going after, but this incident proves the bar for security is not guaranteed to start at certain levels. When you leave your networks wide open with all the goodies on the table, don't be surprised if someone walks off with the food, pooch, car keys, bank cards and chaise longue.

That's enough Internets for me today.

Microsoft Cloud Takeover Ransomware

Ransomware has been a continual thorn in the side of business for several years now and much of the reported incidents (not everyone does still it seems) are related to systems on corporate network fleets. In a continually "cloud-first" narrative for business, it appears that cybercriminal groups are also chanting the "cloud-first" mantra and evolving their attack patterns.

Microsoft's threat intelligence team have caught one of the financially-motivated groups they monitor, Storm-0501, breaking into an enterprise's on-premesis and cloud environments, exfiltrating and destroying resident data before demanding a ransom to get the data back. The attack does not rely on normal malware deployment and saw the enterprise contacted over their own Teams platform to demand the payment.

The attacks seem to pivot from on-premesis systems to the cloud environment through lateral movement and privilege escalation, with the use of Evil-WinRM for remote code execution. DCSync attacked Active Directory and stolen credentials were then used to target cloud-connected systems and ultimately register an Entra ID tenant owned by the criminal group as a trusted federated domain, creating a backdoor to the systems.

The multi-stage processes show a determined and resourceful group with understanding in on-premesis and cloud environments with a new angle on the ransomware process.

Microsoft have made a change to Entra ID that aims to prevent attackers from abusing the Directory Synchronisation Accounts in attacks to escalate privileges. They also advise using the Trusted Platform Module on Entra Connect Sync Server to store sensitive credentials and cryptographic keys, preventing those from being stolen by attackers.

The principles of least-privilege and zero trust should be considered with these hybrid environments, and as always - MFA.

Kill Switch Dev gets 4 years

The end of employment can be a traumatic experience and humans can respond in many ways. The response to their termination displays everything about their character, and for Davis Lu, a 55-year old Chinese national living legally in Houston, it spoke volumes about the vindictive nature of his personality.

Davis worked for an Ohio-based company from 2007 until 2019 and after a corporate restructuring and subsequent demotion in 2018, he began to embed malicious code throughout the corporate Windows environment. His malware included infinite Java thread loops designed to overwhelm servers and crash production systems along with a kill switch named "IsDLEnabledinAD" that would automatically lock all users out of their accounts if Davis's account was disabled in Active Directory.

When Davis's employment was terminated on the 9th September 2019, the kill switch activated and caused thousands of users to be locked out of their systems. The disruption in services caused the company to loose hundreds of thousands of dollars.

Davis was instructed to return his laptop as part of the termination of employment and reportedly deleted encrypted data from his device. Investigators later discovered search queries on the device researching how to elevate privileges, hide processes and quickly delete files.

Having been found guilty of intentionally causing damage to protected computers earlier in the year the sentencing has now landed Davis with a four-year prison sentence and then a three year supervised release following his prison term.

The case again highlights the importance of least-privilege access to systems and auditing. Humans will always react emotionally to events, but the business can often protect itself and other employees with simple controls and processes. Undoubtedly, Davis's case will not be the last we see hitting the news in the current uncertain climate of large business layoffs.

UK Apple Encryption Order Reversal

In a sudden turnaround, the US director of national intelligence has said the UK has withdrawn it's demand to access global Apple user's data if required.

In December 2024, the UK government made the demands to access encrypted data from users worldwide - effectively requesting the breaking of encryption processes and smashing security into pieces on the Apple platforms. The backlash was, as expected, strong and loud. Apple responded with a "We have never built a backdoor or master key to any of our products or services, and we never will" and withdrew ADP from the UK market, starting a legal proceeding to challenge the order.

Whilst you can understand the framing of "we want to ensure criminals cannot hide", the recent enforcement for age verification checks drives the narrative of "if you have nothing to hide you have nothing to fear" and that, my digital friends, is a worrying cyber alleyway we do not want to end up in.

Whether ADP makes it's way back into the UK or the court case proceeds will be one to keep an eye on over the coming months. As usual, no comment has been made in response from the UK government in relation to changes in policy. The US and UK already have law enforcement agreements in place for access to data, so we might find the front door being shut and the back door being opened elsewhere.

Cisco FMC Vulnerability

Another day, another Cisco vulnerability, and this one has a critical 10.0 CVSS rating. If this were a gymnastics competition, they'd be up there on the podium this year.

The CVE-2025-20265 is caused by an improper handling of user input by Firewall Management Center's RADIUS authentication subsystem during the login process. Thankfully, the exploit is only possible if RADIUS authentication is configured for the web-based management interface, SSH mangement or both.

Found during internal security testing by Cisco software engineer Brandon Sakai, the issue is not known to be exploited in the real world (yet!).

Cisco FMC is a central management platform for products such as their firewalls, IDS, URL filtering and anti-malware tools. Attacking this central hub to other devices on the corporate network is a prime target and likely to be actively prodded by foreign government groups.

Cisco warned in their security bulletin "An attacker could exploit this vulnerability by sending crafted input when entering credentials that will be authenticated at the configured RADIUS server, a successful exploit could allow the attacker to execute commands at a high privilege level."

More information on the issue is available on Cisco's Security Advisory page.

HTTP/2 Vulnerability to Crash

The HTTP/2 protocol, introduced in 2012 to provide a range of improvements on the original has become the most widely used web protocol, despite the newer HTTP/3 variant being released in 2019. The introduction of HTTP/2 brought a huge range of improvements on the original, however it also (as with all new software) brought a range of new bugs into the world.

In 2023, the 'Rapid Reset' flaw brought the protocol into the limelight with the ability to DoS servers. Having undergone a patch to correct this flaw, it now appears as if the patch wasn't as complete as one might hope. Security researchers Gal Bar Nahum, Anat Bremler-Barr, and Yaniv Harel have released details of a "common design flaw" in implementations of HTTP/2 allowing the possibility of "massive DoS attacks"

CVE-2025-8671 which extends the previous CVE-2023-44487 "Rapid Reset" vulnerability allows an attacker to bypass concurrency limits which would normally prevent a server from accepting requests for too much work. Rapid Reset allowed the bypass of limits by quickly cancelling a request, however in MadeYouReset they can trick the server into cancelling the request on the client's behalf which bypasses the protection put in place in 2023.

The vulnerability discovery partially supported by Thales' Imperva has further detail and a range of mitigation strategies on their blog.

Even "well-crafted" requests and traffic can be used in weaponised form, it would continue to appear.

Satellite Hacking

Researchers from VisionSpace Technologies displayed a range of vulnerabilities in satellite operations software at BlackHat. The team, having undertaken responsible disclosure beforehand, demoed several major flaws in software used at ground stations and onbobard craft, displaying the posibility of takeover or denial of service.

Space is increasingly becoming a crowded and sensitive operational zone, with the cost of delivering equipment to orbit reducing each year. Communication with orbital craft occurs over RF, with newer equipment undertaking key exchange for encrypted communication. It is possible it seems, however, to crash orbital platforms with an unencrypted message, and then force a rotation of keys.

"So basically, you send a packet to the spacecraft, and the entire software crashes and reboots, which then actually causes the spacecraft, if it's not properly configured, to reset all its keys. And then you have zero keys on the spacecraft that you can use from that stage on.", Milenko Starcik announced.

The issues discovered were not solely related to craft, but also the software used for tasking such as Yamcs and OpenC3Cosmos which allowed a range of cross-site-scripting and take control over satellite equipment.

The team's discovery has provided a range of security improvement to the operation of craft able to be updated, however it is likely that out of the thousands of orbital craft in the sky, many of them will be vulnerable to a range of attack vectors. Space may or may not be the final frontier of cybersecurity, it seems.

PAM Backdoor in Linux

A previously undocumented Linux backdoor dubbed "Plague" has been discovered, which apparently has evaded detection for a year.

Nextron Systems researcher, Pierre-Henri Pezier, noted "The implant is built as a malicious PAM, enabling attackers to silently bypass system authentication and gain persistent SSH access". Plague integrates deeply into the authentication stack, surviving system updates with minimal forensice traces.

Multiple Plague artifacts were found to be uploaded to VirusTotal since July 29th 2024, however none of them were detected as being malicious by the antimalware routines.

Plague has several features:

  • Antidebug, hindering analysis and reverse engineering.
  • String obfuscation, protecting sensitive strings and offsets which complicates detection.
  • Static password, allowing covert access for attackers.
  • Hidden session artifacts, erasing traces of attacker activity.

The Plague backdoor represents a worryingly sophisticated threat to Linux infrastructure, exploiting core auth processes to maintain stealth and persistence on the target system.

Europol Shuts Down Cybercrime Forum

On the 22nd July as a part of a series of coordinated enforcement actions, the French Police and Paris Prosecutor in close cooperation with their Ukrainian counterpart and Europol arrested the suspected administrator of xss.is, one of the world's most influential Russian-speaking cybercrime platforms.

With more than 50,000 registered users, it served as a key marketplace for stolen data, hacking tools and illicit services while being a central platform for some of the most active and dangerous cybercriminal networks.

The arrest in Kyiv, Ukraine, happened after a long investigation dating back to 2021, which was initiated by the French Police. Europol provided essential operational and analytical support throughout the investigation, facilitating coordination and information exchange between the French Police and Ukrainian authorities.

Europol assisted in mapping the cybercriminal infrastructure, linking the suspect to other major threat actors. They deployed a mobile office to support French and Ukrainian teams with on-site coordination and evidence collection. The seized data will be analysed to support ongoing investigations across Europe and beyond it's borders.

The administrator of the forum was known to have a central role in enabling criminal activity alongside acting as the technical operator of the site. He is also believed to have run thesecure.biz private messaging service tailored to the needs of the cybercriminal underground. It's estimated that he made around seven million euros in profit from advertising and facilitation fees.

Sharepoint Actively Exploited

The recently disclosed vulnerabilities for Sharepoint on-premises (CVE-2025-53770, CVE-2025-49705, CVE-2025-53771 and CVE-2025-49706) are now being actively exploited by at least three Chinese groups according to reports from Microsoft's threat team.

Microsoft has observed the nation-state groups Linen Typhoon and Violet Typhoon exploiting the vulnerabilities, targeting Internet-facing Sharepoint servers. The groups are known for specifically focusing on data theft. A third actor is not as-yet known to be state-sponsored, classified as "Storm-2603" is continuing to be monitored.

Administrators are being warned if they have on-premises Sharepoint that has not yet been patched, it is likely the systems have been compromised.

SharePoint Server variants "Subscription Edition", "2019", and "2016" all have fixes available to download.

"In observed attacks, threat actors send a crafted POST request to the SharePoint server, uploading a malicious script named spinstall0.aspx. Actors have also modified the file name in a variety of ways, such as spinstall.aspx, spinstall1.aspx, spinstall2.aspx, etc. The spinstall0.aspx script contains commands to retrieve MachineKey data and return the results to the user through a GET request, enabling the theft of the key material by threat actors."

With the disclosure and active exploits, it's likely smaller or individual groups will begin to utilise the attack vector to gain access to unpatched Sharepoint systems.

Cisco ISE Vulnerabilities

Cisco has issued a patch for the Identity Services Engine (ISE) and ISE Passive Identity Connector (ISE-PIC) that allowed unauthenticated remote attackers the ability to run arbitrary code on the operating system with root privileges.

CVE-2025-20337 is a 10/10 horror-fest related to CVE-2025-20281, disclosed in June and also receiving a 10/10 CVSS rating.

There are no workarounds but thankfully Cisco released a software update that fixes the flaws, along with CVE-2025-20282. According to Cisco there are no known exploits (yet) for any of the vulnerabilities. However, with disclosure comes a slew of "research" by people on both sides of the ethical divide.

Cisco's security advisory website provides details on affected systems and includes:

  • CVE-2025-20281 and CVE-2025-20337: These vulnerabilities affect Cisco ISE and ISE-PIC releases 3.3 and 3.4, regardless of device configuration. These vulnerabilities do not affect Cisco ISE and ISE-PIC Release 3.2 or earlier.
  • CVE-2025-20282: This vulnerability affects only Cisco ISE and ISE-PIC Release 3.4, regardless of device configuration. This vulnerability does not affect Cisco ISE and ISE-PIC Release 3.3 or earlier.

Given the nature of the vulnerability, it's critical to run patching as soon as possible now the disclosure is freely available.
