More n8n Woes

13th January 2026

Obviously in the crosshairs of nefarious groups, n8n continues to suffer further woes with a discovery of supply chain attacks on eight packages on npm registry. Endor Labs have uncovered the first in a new wave of supply chain attacks on n8n and its community node ecosystem.

Behaving like any other n8n integration once it is installed, it displays the usual configuration screens and saves the Google account OAuth tokens in encrypted format to the n8n credential store. When the workflow is executed, however, it runs code to decrypt the stored tokens using n8n's master key and then exfiltrates the credentials to a remote server.

The list of currently malicious packages is as follows:

  • n8n-nodes-hfgjf-irtuinvcm-lasdqewriit (4,241 downloads, author: kakashi-hatake)
  • n8n-nodes-ggdv-hdfvcnnje-uyrokvbkl (1,657 downloads, author: kakashi-hatake)
  • n8n-nodes-vbmkajdsa-uehfitvv-ueqjhhhksdlkkmz (1,493 downloads, author: kakashi-hatake)
  • n8n-nodes-performance-metrics (752 downloads, author: hezi109)
  • n8n-nodes-gasdhgfuy-rejerw-ytjsadx (8,385 downloads, author: zabuza-momochi)
  • n8n-nodes-danev (5,525 downloads, author: dan_even_segler)
  • n8n-nodes-rooyai-model (1,731 downloads, author: haggags)
  • n8n-nodes-zalo-vietts (4,241 downloads, authors: vietts_code and diendh)

Previous warnings on community node security risks have been issues by n8n and on self-hosted n8n instances it is strongly advised to disable community nodes by setting N8N_COMMUNITY_PACKAGES_ENABLED to false.

It is likely automation systems such as n8n along with CI/CD tooling will increasingly come under attack as workloads on them integrate with so many different platforms and hold high permission credentials. It is a reminder that security boundaries are not just around running platforms, but encircle the whole developer and operational environment.
