LiteLLM Compromised

26th March 2026

Python Package Index have removed two versions of LiteLLM, the open source interface for accessing multiple large language models after a supply-chain attack injected them with credential-stealing code.

LiteLLM v1.82.7 and v1.82.8 have been removed due to malware in the litellm_init.pth file. The compromise appears to have originated form the use of Trivy in the project's CI/CD pipeline. According to details from Aqua Security, the attackers took advantage of a misconfiguration in Triby's GitHub Actions environment to steal a privileged access token allowing the manipulation of CI/CD, and the subsequent push of malicious Trivy versions ( v0.69.5 and v0.69.6 ) as DockerHub images.

The advisory warns anyone who has installed and run the project should assume any credentials available to the LiteLLM environment have been exposed and should be revoked or rotated immediately.
