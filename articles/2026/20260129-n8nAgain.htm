n8n Again

29th January 2026

It's been a week, it's time for another notification of security issues with n8n, proving that sandboxing environments is hard, and secure automation is still struggling to keep up with language changes and weirdisms.

CVE-2026-1470 (9.9 CVSS) and CVE-2026-0863 (8.5 CVSS) have been discovered and responsibly disclosed by the JFog team and causes detailed eloquently on their site.

CVE-2026-1470 relates to the expression feature that allows teams to set node parameters dynamically. You can execute JavaScript when encasing in parenthesis which is then supposed to rely on the Abstract Syntax Tree (AST)-based sandbox to validate the inherantly dangerous process of executing code. However, it would seem that you can't keep baby in the corner JavaScript from running amok whenever you use the "WITH" statement.

CVE-2026-0863 relates to the Python Code Node allowing users to execute Python code for processing. Again, an AST sandbox is employed to try to protect the crown jewels keep things safe, however it would appear that under error throw conditions it's possible to escape the prison and interact with the system.

Fun stuff, just as I was thinking of playing with n8n this year. Guess I'll just break out the 8-bit microcomputers instead and disappear to a happier time in the land of computing.
