PAM Backdoor in Linux

3rd August 2025

A previously undocumented Linux backdoor dubbed "Plague" has been discovered, which apparently has evaded detection for a year.

Nextron Systems researcher, Pierre-Henri Pezier, noted "The implant is built as a malicious PAM, enabling attackers to silently bypass system authentication and gain persistent SSH access". Plague integrates deeply into the authentication stack, surviving system updates with minimal forensice traces.

Multiple Plague artifacts were found to be uploaded to VirusTotal since July 29th 2024, however none of them were detected as being malicious by the antimalware routines.

Plague has several features:

  • Antidebug, hindering analysis and reverse engineering.
  • String obfuscation, protecting sensitive strings and offsets which complicates detection.
  • Static password, allowing covert access for attackers.
  • Hidden session artifacts, erasing traces of attacker activity.

The Plague backdoor represents a worryingly sophisticated threat to Linux infrastructure, exploiting core auth processes to maintain stealth and persistence on the target system.
