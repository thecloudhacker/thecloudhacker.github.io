MongoDB Bleeds Memory Data

28th December 2025

A flaw in Zlib compression protocol header length ( CVE-2025-14847 ) which is utilised by default in MongoDB can return uninitialized heap memory without authenticating to the server.

Tracked by MongoDB and thankfully with a patch available ( MongoDB Jira Ticket), the MongoDB team urge users to patch immediately even though there are no current reports of active exploitation.

This data leak vulnerability affects the following versions:

  • MongoDB 8.2.0 through 8.2.3
  • MongoDB 8.0.0 through 8.0.16
  • MongoDB 7.0.0 through 7.0.26
  • MongoDB 6.0.0 through 6.0.26
  • MongoDB 5.0.0 through 5.0.31
  • MongoDB 4.4.0 through 4.4.29
  • All MongoDB Server v4.2 versions
  • All MongoDB Server v4.0 versions
  • All MongoDB Server v3.6 versions

The team have also advised "If you cannot upgrade immediately, disable zlib compression on the MongoDB Server by starting mongod or mongos with a networkMessageCompressors or a net.compression.compressors option that explicitly omits zlib. Example safe values include snappy,zstd or disabled"
