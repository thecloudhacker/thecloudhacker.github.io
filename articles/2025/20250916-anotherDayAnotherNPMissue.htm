Another Day, Another NPM Supply Chain Attack

16th September 2025

Ah yes, it's another day ending in "y" so it must be time for another NPM supply chain attack.

Today's list of affected NPM packages is quite extensive and includes some code (bundle.js injected into each package) that tries to search for credentials by pulling and using TruffleHog. A great rundown was provided by Step Security

The new compromise is much more convoluted, and has self-propagation included to further infect packages of the maintainer utilising the infected package.

Yet another day that I'm having to run scanning of my company software bill of materials lists. Stay safe out there - I'm expecting more of the same as time passes with increasingly complex malware bundled into the supply chain.
