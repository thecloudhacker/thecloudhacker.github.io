NPM SBoM check System ( Part 2 )

19th September 2025

On the back of the recent frantic dev process to verify our Software Bill of Materials, I ended up rejigging some hacked together code and providing it on my Github Site in the hope it's of use to others.

The codebase is a quick and dirty hack, so don't start hosting it on external endpoints as I've not run any testing against it and I know there's some older packages on there.

You will need to generate your own SBOM files before using the process, there's a bunch of tooling to provide that for you available online. Once you have that, then throw the sbom at the searcher.

Anyhow... there we have it. This was a week that was. I'm sure there will be plenty more weeks like this in the months and years ahead.

Stay safe out there, friends.
