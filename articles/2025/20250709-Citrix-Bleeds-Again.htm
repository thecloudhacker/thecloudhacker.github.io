Citrix Bleeds Again

9th July 2025

Citrix Netscaler has been in the feeds again this week with two vulnerabilities being actively exploited in the world wild web.

CVE-2025-5777 and CVE-2025-6543 are proving that Arnie was right in Predator - "If it bleeds, we can kill it". Well, perhaps not kill the Netscaler, but make it hemorrhage data from memory.

CVE-2025-5777 is particularly fun according to Watchtowr's lab analysis, showing that simple request replays with crafted input parameters can spit memory contents back at you.

It appears that the Citrix team potentially have uninitialised local variables in the authentication processing which is handing back random pieces of data from memory. Whilst it's important to note that you cannot request specific memory locations, the repeated requests to a live, active system could yeild important information from other users on VPN connections.

Defensive coding practices are designed to mitigate against this sort of issue, ensuring variables are nulled and memory pointers aren't left opening doors to data.

As always, check your patch status and stem the bleeding.
